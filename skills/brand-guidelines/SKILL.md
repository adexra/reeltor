# Adexra Brand Styling

## Overview

Adexra’s design system is built around a **dark, high-contrast, product-first interface**.

It is designed to feel:
- fast
- precise
- technical
- controlled

This is not a “friendly brand”.
This is a **performance system engineered for conversion**.

**Keywords**: dark UI, SaaS interface, system design, conversion design, Adexra, product-first UI

---

## Brand Guidelines

### Core Colors

**Base Layer:**
- Background: `#01020E`
- Foreground (text): `#F4F4F6`

**Surface Layers:**
- Muted Background: `#0D0F1E`
- Border: `rgba(244, 244, 246, 0.07)`
- Strong Border: `rgba(244, 244, 246, 0.14)`

**Accent System (Primary Action Driver):**
- Accent Blue: `#3362FF`
- Accent Soft: `rgba(51, 98, 255, 0.12)`
- Accent Border: `rgba(51, 98, 255, 0.3)`

**Feedback Colors:**
- Error: `#FF3B5C`
- Error Soft: `rgba(255, 59, 92, 0.10)`

**Text Hierarchy:**
- Primary: `#F4F4F6`
- Secondary: `#6B7080`

---

## Contrast System (Critical)

Adexra does **not** rely only on dark UI.

It uses **controlled contrast zones** to guide attention.

### Light Surfaces (Strategic Use Only)

**Allowed:**
- Soft White: `#F4F4F6`
- Pure White: `#FFFFFF` (limited use)

### Purpose

Light surfaces exist to:
- highlight key information
- isolate important content
- create visual breaks
- increase readability
- drive attention to decisions

### Usage Rules

- Never use full white backgrounds across entire pages
- Always maintain dark surroundings
- Use for sections, cards, or focus blocks only
- Prefer off-white over pure white when possible

### Supporting Styles

- Subtle borders:
  `rgba(0,0,0,0.05)`
- Minimal shadows (very soft, almost invisible)

## Design Insight

Dark UI improves:
- focus
- perceived performance
- visual hierarchy

Contrast improves:
- attention control
- readability
- conversion rates

Adexra combines both:
**dark environment + light focus + blue action**

---

## Typography

- **Primary Font**: Monospace (system or variable)
- **Display Font**: Serif (used selectively)

### Rules

- Monospace for UI, labels, structure
- Serif for headlines or brand emphasis
- Tight spacing, high readability
- No decorative fonts

---

## Design Principles

### 1. System Over Decoration

- Everything must feel modular
- No random styling
- Reusable components only

---

### 2. Contrast = Hierarchy

Hierarchy is created through:
- brightness
- opacity
- spacing

Not visual noise.

---

### 3. Motion with Purpose

Use motion to:
- guide attention
- provide feedback
- enhance perceived speed

Avoid:
- slow animations
- decorative effects

---

### 4. Precision Layout

- Grid-based system (60px rhythm)
- Consistent spacing
- Clean alignment at all times

---

## UI Elements

### Cursor

- Hidden default cursor
- Custom interaction layer

---

### Backgrounds

**Noise Layer**
- Very subtle texture
- Prevents flat appearance

**Grid Overlay**
- Suggests structure
- Used sparingly

---

### Scrollbar

- Minimal (3px width)
- Accent-colored thumb
- Clean, unobtrusive

---

### Selection

- Accent blue highlight
- High contrast text

---

## Effects & Motion

### Accent Glow

- Applied to key elements only
- Reinforces importance

---

### Animations

- Ticker (continuous motion)
- Gradient drift (ambient movement)
- Fade-up (section entry)
- Error pulse (feedback)

All motion must feel:
- fast
- smooth
- intentional

---

## Component Behavior

### Sections

- Fade-in on scroll
- Slight upward motion

---

### Interactive Elements

- Instant feedback
- Subtle hover states
- No delays

---

## What NOT to Do

- ❌ Full bright interfaces
- ❌ Multiple competing accent colors
- ❌ Heavy shadows or “card-heavy” UI
- ❌ Cluttered layouts
- ❌ Slow or decorative animations

---

## Positioning Through Design

Adexra should feel like:

> “This system is engineered, not designed.”

This increases:
- trust
- perceived value
- conversion potential
